# v0.3.0 — HSK Standard Course Textbook Integration
Version: 0.3.0 | Status: Design | Date: 2026-03-17

> Integrate the official HSK Standard Course textbook structure into the app.
> Every lesson from the textbook becomes a structured learning unit: dialogues,
> grammar, pinyin, characters, cultural notes, and targeted exercises.
> A new lesson-based navigation path sits alongside the existing SRS quiz flow.

---

## The Problem

The current app is a vocabulary-first SRS trainer. It works, but it has no connection to the structure learners are actually following in class or self-study. The HSK Standard Course is the most widely used official curriculum — 15 lessons per level, each with a clear pedagogical arc. Learners using that textbook have no way to tie what they study in the app to what they study in the book.

Gaps today:
1. **No lesson structure.** Words exist in a flat level list, not organized by the chapter they're introduced in.
2. **No dialogue practice.** The textbook's 课文/Text section (dialogues) is the core reading/listening input — not modeled at all.
3. **No grammar exercises.** `GrammarPoint` rows exist in the schema but nothing is populated or surfaced to users.
4. **No pinyin lessons.** Beginners need structured pinyin introduction (initials, finals, tones) — missing entirely.
5. **No character lessons.** Stroke order and character components for beginners are in the textbook but not the app.
6. **No cultural notes.** The textbook includes cultural context every 5 lessons — absent from the app.

---

## Validation Stories

> **Story 1 — "I can follow the book in the app"**
>
> A student is on HSK1 Lesson 3. They open the app, navigate to Lessons → HSK 1 → Lesson 3.
> They see the lesson's new vocabulary, read through the dialogue, review the grammar note,
> and do the lesson exercises. It mirrors exactly what chapter they're reading in the textbook.

> **Story 2 — "I can practice just this lesson's words"**
>
> From Lesson 3's page, the student taps "Practice this lesson".
> The quiz loads with only the vocabulary from Lesson 3, SRS-ordered as usual.
> After finishing, XP is awarded and progress is saved the same as a regular session.

> **Story 3 — "I can read the dialogue and test myself"**
>
> The student opens the Dialogue tab of Lesson 3. They see the two dialogues with Chinese,
> pinyin, and English translation. They can toggle pinyin off to test themselves.
> A "Dialogue Quiz" button generates fill-in-the-blank or translation exercises from the dialogue lines.

> **Story 4 — "New learner gets a structured start"**
>
> A complete beginner picks HSK1 and the app suggests starting from Lesson 1.
> They see the pinyin introduction before any vocabulary. Characters are introduced
> with stroke order diagrams. The onboarding path mirrors the textbook's pedagogical sequence.

> **Story 5 — "Content loads as new chapters are added"**
>
> The developer adds Lessons 9–15 for HSK1 and all of HSK2 by running a seed script.
> No code changes required — the lesson route handles any lessonId dynamically.

## Architecture Decision

### Approach: Lesson as content hub, SRS stays unchanged

The SRS quiz system (`/practice/quiz`) remains untouched. Lessons add a parallel path:
- **SRS path:** random-ish, SRS-ordered, level-wide vocabulary practice (existing)
- **Lesson path:** structured, chapter-by-chapter, textbook-aligned (new)

These are complementary. The lesson path seeds vocabulary into the SRS; the SRS path reinforces it over time. Completing a lesson quiz upserts `UserWordProgress` the same way the regular quiz does.

### Data model strategy

Extend the existing schema minimally:
- Add a `Lesson` model as the central hub
- Add content models hanging off `Lesson` (dialogues, pinyin, characters, culture)
- Extend `HskWord` with an optional `lessonId` (which lesson first introduces this word)
- Extend `GrammarPoint` with an optional `lessonId`
- Extend `Exercise` with an optional `lessonId` (for lesson-specific exercises)

This keeps `HskWord` as the vocabulary source of truth. The lesson just groups words contextually.

## Data Model Changes

### New: `Lesson`

```prisma
model Lesson {
  id            String              @id @default(uuid())
  level         Int                 // HSK level 1–6
  lessonNumber  Int                 // 1–15 within the level
  title         String              // Chinese title e.g. "你好"
  titlePinyin   String              // e.g. "Nǐ hǎo"
  subtitle      String              // English subtitle e.g. "Hello"
  objectives    String[]            // What you'll learn
  createdAt     DateTime            @default(now())

  words         HskWord[]           // words introduced in this lesson
  dialogues     LessonDialogue[]
  grammarPoints GrammarPoint[]
  pinyinDrills  LessonPinyinDrill[]
  characters    LessonCharacter[]
  cultureNote   CultureNote?
  exercises     Exercise[]
  userProgress  UserLessonProgress[]

  @@unique([level, lessonNumber])
  @@map("lessons")
}
```

### New: `LessonDialogue`

Stores the textbook's 课文 dialogues. Each lesson has 2–3 dialogues.
Each dialogue has turns: `[{ speaker, chinese, pinyin, english }]`.

```prisma
model LessonDialogue {
  id              String   @id @default(uuid())
  lessonId        String
  dialogueNumber  Int      // 1, 2, 3 within lesson
  title           String?
  turns           Json     // [{ speaker, chinese, pinyin, english }]
  lesson          Lesson   @relation(fields: [lessonId], references: [id])

  @@map("lesson_dialogues")
}
```

### New: `LessonPinyinDrill`

The textbook's 拼音 section introduces initials, finals, and tones progressively.

```prisma
model LessonPinyinDrill {
  id        String   @id @default(uuid())
  lessonId  String
  type      String   // "initial" | "final" | "tone" | "combination"
  content   Json     // [{ symbol, examples: [{ pinyin, character, meaning }] }]
  order     Int
  lesson    Lesson   @relation(fields: [lessonId], references: [id])

  @@map("lesson_pinyin_drills")
}
```

### New: `LessonCharacter`

The textbook's 汉字 section. Links to the underlying `HskWord` when applicable.

```prisma
model LessonCharacter {
  id                     String   @id @default(uuid())
  lessonId               String
  character              String
  wordId                 String?
  strokeCount            Int?
  strokeOrderUrl         String?
  componentsExplanation  String?
  order                  Int
  lesson                 Lesson   @relation(fields: [lessonId], references: [id])
  word                   HskWord? @relation(fields: [wordId], references: [id])

  @@map("lesson_characters")
}
```

### New: `CultureNote`

Cultural notes appear every 5 lessons (lessons 5, 10, 15 per level).

```prisma
model CultureNote {
  id        String   @id @default(uuid())
  lessonId  String   @unique
  title     String
  content   String
  imageUrl  String?
  lesson    Lesson   @relation(fields: [lessonId], references: [id])

  @@map("culture_notes")
}
```

### New: `UserLessonProgress`

Track which lessons a user has started/completed.

```prisma
model UserLessonProgress {
  id          String    @id @default(uuid())
  userId      String
  lessonId    String
  startedAt   DateTime  @default(now())
  completedAt DateTime?
  user        User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  lesson      Lesson    @relation(fields: [lessonId], references: [id])

  @@unique([userId, lessonId])
  @@map("user_lesson_progress")
}
```

### Modified: `HskWord`

Add optional `lessonId` to mark which lesson first introduces this word.

```prisma
// Add to HskWord:
lessonId      String?
lesson        Lesson?   @relation(fields: [lessonId], references: [id])
```

### Modified: `GrammarPoint`

Add optional `lessonId`.

```prisma
// Add to GrammarPoint:
lessonId      String?
lesson        Lesson?   @relation(fields: [lessonId], references: [id])
```

### Modified: `Exercise`

Add optional `lessonId` for lesson-specific exercises (from the textbook's 练习 section).

```prisma
// Add to Exercise:
```

## Navigation & Routing

### New routes

| Route | Page | Description |
|-------|------|-------------|
| `/learn/lessons` | Lesson hub | HSK level picker + progress overview |
| `/learn/lessons/hsk[level]` | Level lesson list | All 15 lessons for a level, with completion badges |
| `/learn/lessons/hsk[level]/[number]` | Lesson detail | Tabbed lesson content (see below) |

### Lesson detail page tabs

Each lesson page has these tabs (only show tabs with content seeded):

1. **Overview** — lesson title, objectives, new word count, grammar points count, completion status
2. **Vocabulary** — new words introduced in this lesson (character + pinyin + definition), "Practice" button
3. **Dialogues** — textbook dialogues with pinyin toggle, "Dialogue Quiz" button
4. **Grammar** — grammar notes from 注释, pattern + examples
5. **Pinyin** — initials/finals/tone drills (HSK1 early lessons only)
6. **Characters** — character breakdown with stroke count and component notes
7. **Culture** — cultural note (only on lessons 5, 10, 15)

### Navigation integration

Add "Lessons" to the sidebar nav alongside the existing "Learn" and "Practice" items.

```
Sidebar:
  Dashboard
  Practice (existing — SRS quiz)
  Lessons     ← new
  Learn       (existing — browse vocabulary)
  Leaderboard
  Profile
```

### Level/lesson URL scheme

Use `hsk1`–`hsk6` slugs: `/learn/lessons/hsk1/3` for HSK1 Lesson 3.
Simple, bookmarkable, matches the textbook's own naming.

## Lesson Quiz Mode

Clicking "Practice this lesson" from a lesson page routes to:
```
/practice/quiz?lessonId=<id>&level=<level>
```

The exercises API (`GET /api/exercises`) already accepts `level`. Extend it to accept `lessonId`:
- When `lessonId` provided, scope word selection to `HskWord.lessonId = lessonId`
- SRS ordering still applies within that scoped set
- Falls back to all level words if lesson has fewer words than requested count

Session completion posts to `/api/sessions` with `lessonId` added.
On completion, mark `UserLessonProgress.completedAt` if all lesson words have been attempted.

## Content Seeding Strategy

All textbook content is seeded via TypeScript seed scripts, not hardcoded in the schema. This makes it easy to add new chapters without touching application code.

### Seed file structure

```
prisma/
  seed/
    lessons/
      hsk1/
        lesson-01.ts   ← vocabulary, dialogues, grammar, pinyin, characters
        lesson-02.ts
        ...
        lesson-15.ts
      hsk2/
        lesson-01.ts
        ...
    seed-lessons.ts    ← master runner: imports all lesson files and upserts
```

### Per-lesson seed shape

```typescript
// prisma/seed/lessons/hsk1/lesson-01.ts
export const lesson: LessonSeed = {
  level: 1,
  lessonNumber: 1,
  title: "你好",
  titlePinyin: "Nǐ hǎo",
  subtitle: "Hello",
  objectives: ["Greet someone", "Ask someone's name", "Introduce yourself"],

  vocabulary: ["nǐ hǎo", "nǐ", "hǎo", "nín", "lǎoshī", "tóngxué"],
  // ^ matched to HskWord.pinyin to set lessonId

  dialogues: [
    {
      dialogueNumber: 1,
      turns: [
        { speaker: "大卫", chinese: "你好！", pinyin: "Nǐ hǎo!", english: "Hello!" },
        { speaker: "玛丽", chinese: "你好！", pinyin: "Nǐ hǎo!", english: "Hello!" },
      ]
    }
  ],

  grammarPoints: [
    {
      title: "你好 as a greeting",
      explanation: "...",
      pattern: "你好",
      examples: [...]
    }
  ],

  pinyinDrills: [
    { type: "initial", content: [{ symbol: "b", examples: [...] }] }
  ],

  characters: [
    { character: "你", strokeCount: 7, componentsExplanation: "亻+ 尔" },
    { character: "好", strokeCount: 6, componentsExplanation: "女 + 子" },
  ]
}
```

Run with: `npx tsx prisma/seed/seed-lessons.ts`
Idempotent (upsert by `[level, lessonNumber]`). Safe to re-run when content is updated.

## Extensibility Plan

### Adding remaining HSK1 chapters (9–15)

1. Create `prisma/seed/lessons/hsk1/lesson-09.ts` through `lesson-15.ts`
2. Run `npx tsx prisma/seed/seed-lessons.ts`
3. Done. No code changes.

### Adding HSK2–6

1. Create `prisma/seed/lessons/hsk[level]/lesson-[nn].ts` files
2. Run seed script
3. Done. The level picker and lesson list routes are already dynamic on `level` param.

### Adding new content types (e.g., audio)

`LessonDialogue.turns` is `Json` — can add `audioUrl` per turn without a schema migration.
`LessonCharacter` has `strokeOrderUrl` ready for stroke-order animation assets.

---

## What This Is NOT

- Not replacing the SRS quiz. The two paths are additive.
- Not a full textbook reader / PDF viewer. Content is structured data, not scanned pages.
- Not an audio player (yet). `audioUrl` fields are reserved but audio is in the P1 backlog.
- Not a writing practice system. Stroke order images are static diagrams, not interactive.
- Not generating exercises automatically from dialogue text (v0.3.0 scope). Exercises come from the seed data. Generative exercise creation from dialogue is a future enhancement.

## Known Gaps / Follow-up

- **Dialogue Quiz not in v0.3.0 scope.** The "Dialogue Quiz" button is designed but exercise generation from dialogue lines is deferred. Button can be stubbed with "Coming soon".
- **Application (运用) section not modeled.** The textbook's applied practice activities are contextual and harder to structure. Deferred.
- **No warm-up (热身) model.** Warm-up activities are pedagogical scaffolding, not easily digitized. Deferred.
- **Full SM-2 wiring.** Sessions API still uses simplified intervals (v0.2.0 known gap). Still deferred.
